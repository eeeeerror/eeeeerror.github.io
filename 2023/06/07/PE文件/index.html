# PE 文件

# 基本结构

Dos 头到节区头是 PE 头的部分，其下的节区是 PE 头部分合称 PE 体

文件的偏移，内存中使用使用 VA（虚拟地址）来表示位置，文件加载到内存时，情况会发生变化，文件的内容一般分为代码（.text), 数据（data），资源节（rsrc）。

# VA&RVA

VA：虚拟内存地址

RVA：相对虚拟地址即相对于基地址的偏移地址

RVA+ImageBase=VA

使用这种的方法原因：PE 文件被加载到进程时（特别是 dll），加载到进程虚拟内存时的特定位置时，有可能已经加载了其他的文件，此时必须要使用重定位将其加载到空白位置，如果 PE 文件使用的是 VA 这无法正常访问，如果是 Rva 只要基准地址的相对位置不变，就能正常访问到指定信息。

# PE 头

# DOS 头

为了考虑兼容性 PE 文件对 DOS 头

在 PE 头前添加了–IMAGE_DOS_HEADER–的结构体来扩展

# IMAGE_DOS_HEADER

1.e_magic:DOS 签名（4D5A=>MZ）

2.e_lfanew: 指示 NT 头的偏移

# DOS 存根

在 DOS 头的下方，是可选项，且大小不固定（即使没有也能够正常运行）

是由数据和代码混合而成

# NT 头：文件头
# IMAGE_FILE_HEADER:

1.Machine

每个 CPU 都有唯一的 Machine 码

2.NumberOfSections

用来指出文件中存在的节区数量，该值一定要大于 0，且若于实际不符合会无法运行

3.SizeOFOptionalHeader

IMAGE_FILE_HEADER 结构体的最后一位成员为 IMAGE_OPTIONAL_HEADER32 的结构体 SizeOFOptionalHeader 成员用来指出长度（IMAGE_OPTIONAL_HEADER32 为 c 语言编写但是 windos 在编译的过程中需要查看 SizeOFOptionalHeader 了明确大小。

4.Characteristics

用于标识文件的属性，文件是否处于可运行的形态，是否为 DLL 文件等信息，以 bit OR 形式组合起来

# NT 头，可选头
# IMAGE_OPTIONAL_HEADER

1.Magic

为 IMAGE_OPTIONAL_HEADER32 时 Magic 码为 10B:IMAGE_OPTIONAL_HEADER64 时 Magic 码为 20B

2.AddressOfEntryPoint

AddressOfEntryPoint 持有 Ep 的 RVA 的值他指出程序最先的代码起始地址，相当重要

3.ImageBase

他指出文件被优先装入的地址，一般来说 PE 文件执行时 PE 装载器，再将文件载入内存，然后把 EIP 的值设置为 ImageBase+.AddressOfEntryPoint

4.SectionAlignment，FlieAlignment

Pe 文件的 BODy 部分划分为若干个区域，FlieAlignment 指定了节区在磁盘文件中最小单位，SectionAlignment 指定了节区在内存中最小单位

5.SizeOfimage

指定 PE image 在虚拟内存中所占有的空间大小，一般而言文件大小与加载到内存的大小不一样

6.SizeOfHeader

SizeOfheader 用来指出整个 PE 头的大小，该值也必须是 FlieAlignment 的倍数，第一节区所在的位置与 SizeOfHeader 的距文件的偏移量是一样的

7.Subsystem

子系统 驱动程序 (1) 图形界面 (2) 控制台、DLL (3)

用来区分系统驱动文件（sys）与普通可执行文件（exe，dll）

8.NumberOfRvaAndSizes

NumberOfRvaAndSizes 用来识别 DataDirectory 数组的个数，PE 装载器通过查看 NumberOfRvaAndSizes 的值来识别数组大小

9.DataDirectory

DataDirectory 是由 IMAGE_DATA_DIRECTORY 结构体组成的数组，数组每项都有被定义的值
