# SMC

可以看成两段程序，smc 解密在上先运行对后面加密的程序的机器码解码，然后在运行后面的解码后的程序。

有指针指向机器码开头，运行就会赋值机器码去加密

对机器码事先加密，刚进入 ida 是看不见，只有运行过后才能看见真正的机器码，ida 翻译会报错，

原理：用前一段代码去解码后一段加密的机器码

# 时间检测器

运行时间和调试时间不一样，在起点设置记录器在终点也一样，然后相减

异或：相同为 0，不同为 1

0xCC：软件断点的原理：调试器在对应设置断点的位置上修改该地址的字节为 0xCC。若是关键位置检测到该指令，可以判断进程处于调试状态。

0xCC 既可以是 INT 3 指令，也可以是其他指令的操作数。

int3 运行会触发异常，会暂停

# 比较校验和

加所有机器码会有一个校验和，如果修改就会不一样

# 线程结构体

断点记录在这个结构体里面

句柄：类似指针

# 故意触发异常

使用 loadlibrary 去故意传入没有的 dll 文件，让他返回 0 去混淆视听，去写分支去阻止反调试
