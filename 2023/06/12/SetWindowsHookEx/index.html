# SetWindowsHookEx

是一个 Windows API 函数，可以用于在 Windows 系统中安装鼠标和键盘的钩子函数。它可以捕获鼠标和键盘事件，并允许开发者在这些事件发生时执行自定义逻辑。通过使用 SetWindowsHookEx，开发人员能够编写更高级的应用程序或工具，如键盘记录器、宏工具和自动化脚本等。此外，SetWindowsHookEx 还可用于监视用户活动以及保护系统安全和隐私。但需要注意的是，如果 Hook 函数的执行时间过长，将会导致系统性能下降，因此必须小心使用。

HHOOK 是一个句柄类型，用于表示钩子过程的句柄。HHOOK 类型的变量被声明为指向 HOOKPROC 函数的指针，以便在 SetWindowsHookEx 函数中使用。返回值为 HHOOK 的函数包括 SetWindowsHookEx 和 SetWinEventHook 等。HHOOK 类型的变量可以用于 UnhookWindowsHookEx 函数卸载钩子。在 Windows 系统中，钩子是一种机制，允许应用程序在事件发生时监视、拦截和响应系统级事件（例如键盘和鼠标事件）。使用 HHOOK 类型的变量，可以管理不同类型的钩子，如键盘钩子、鼠标钩子、日志文件记录钩子等。需要注意的是，HHOOK 类型的变量只有在成功安装钩子时才有意义，因此需要检查 SetWindowsHookEx 函数的返回值是否为 NULL。

GetCurrentThreadId () 是一个 Windows API 函数，用于获取当前线程的标识符（ID）。线程 ID 是一个唯一的 32 位整数，可用于在多线程程序中区分不同的线程。GetCurrentThreadId () 函数没有参数，直接调用即可返回当前线程 ID。在线程中安装钩子时，需要指定与钩子关联的线程 ID，可以使用 GetCurrentThreadId () 获取当前线程 ID，也可以使用 GetWindowThreadProcessId () 函数获取与指定窗口关联的线程 ID。在使用 SetWindowsHookEx 函数安装全局钩子时，需要注意钩子处理程序应该在一个单独的线程中运行，以便避免阻塞其他线程的操作。因此，在使用 SetWindowsHookEx 安装全局钩子时，可以使用 CreateThread 函数创建一个新线程，并将钩子处理程序作为其入口点。

int nCode, WPARAM wParam, LPARAM lParam

2023/4/25 21:28:29

int nCode、WPARAM wParam 和 LPARAM lParam 是钩子处理程序的参数，用于获取系统传递给钩子处理程序的信息。具体来说，int nCode 表示当前事件在钩子处理程序中的编码，通过判断 nCode 是否小于 0 可以确定事件是要被处理还是要被传递给下一个钩子；WPARAM wParam 和 LPARAM lParam 则分别表示与当前事件相关的数据，其类型和含义取决于钩子的类型和事件的类型。对于鼠标钩子来说，wParam 表示鼠标事件的类型（例如 WM_LBUTTONDOWN），lParam 包含了鼠标事件的详细信息（例如鼠标的位置信息）；对于键盘钩子来说，wParam 表示虚拟键码或者硬件扫描码，lParam 包含了键盘事件的详细信息（例如扫描码）。在钩子处理程序中，可以通过这些参数获取事件相关的信息，并根据需要进行处理或者传递给下一个钩子。需要注意的是，在钩子处理程序内部调用 Windows API 函数时，可能会影响系统的性能，应当尽量避免这种情况的发生。
